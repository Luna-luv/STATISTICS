## (1) 탐색적 데이터 분석 EDA
### EDA 주요 목적
- 데이터의 형태와 척도가 분석에 알맞게 되어있는가(sanity checking)
- 데이터의 평균, 분산, 분포, 패턴 등의 확인을 통해 데이터 특성 파악
- 데이터의 결측값이나 이상치 파악 및 보완
- 변수 간 관계성 파악
- 분석 목적과 방향성 점검 및 보정

### EDA 과정 코드 정리
- 필요한 패키지 import
```python
import seaborn as sns
import pandas as pd
```

- 데이터 샘플 확인
```python
df.head()
```

- 칼럼 속성 확인
```python
df.info()
```
컬럼명, 결측값개수, 데이터타입 파악 가능

- 칼럼 통계치 확인
```python
df.describe()
```
숫자형 변수들의 기술통계적 측정치

- 각 칼럼의 왜도, 첨도 확인
```python
df.skew()
df.kurtosis()
```

## 2. 공분산과 상관성 분석
### 공분산
(1) 정의 
- 두 변수의 **동시 변동 정도**를 수치로 표현
- 두 변수의 값이 평균에서 얼마나 벗어나는지 확인

(2) 해석
- 양(음)의 공분산 : X증가(감소) -> Y증가(감소)
- 0에 가까움 : 거의 무관

### 상관계수
(1) 정의
- 공분산을 **정규화**
- -1 ~ +1 사이의 값을 가지며, 방향성과 강도 표현현
- ⚠️기울기와 상관계수는 관계 없음

(2) 변수 척도별 상관계수
| 상관계수 종류        | 적용 척도             |
| -------------- | ----------------- |
| Pearson        | 간격/비율 – 간격/비율     |
| Spearman (서열형) | 서열 – 서열           |
| Point-Biserial | 간격 – 명목(이분형)      |
| Phi            | 명목(이분형) – 명목(이분형) |
| Cramer’s V     | 명목 – 명목 (2×2 이상)  |

### 공분산 vs 상관계수
| 항목     | 공분산            | 상관계수          |
| ------ | -------------- | ------------- |
| 값의 범위  | 제한 없음          | -1 \~ 1       |
| 크기 해석  | 불가능 (단위 영향 큼)  | 가능 (무단위)      |
| 정규화 여부 | X              | O             |
| 관계 해석  | 방향만 판단         | 방향 + 강도 판단    |
| 사용 목적  | 선형관계의 존재 여부 파악 | 선형관계 강도/방향 파악 |

## 3. 시간 시각화
### 선그래프 vs 막대그래프
| 항목           | **선그래프 (Line Chart)**          | **막대그래프 (Bar Chart)**         |
| ------------ | ------------------------------ | ----------------------------- |
| **용도**       | 시간 흐름에 따른 **연속적 변화** 시각화       | 시간 흐름에 따른 **구간별 값 비교**        |
| **적합한 데이터**  | **시간 간격이 촘촘한 데이터** (일별, 시간별 등) | **시간 간격이 넓은 데이터** (월별, 연도별 등) |
| **표현 방식**    | 점을 선으로 연결해 **추세** 강조           | 막대의 길이로 **값의 크기** 강조          |
| **추세선**      | **이동평균선** 등 부드러운 패턴 표현 가능      | 추세선 표현에 적합하지 않음               |
| **세부 항목 표현** | 선의 색상 또는 선 개수로 구분              | **누적 막대그래프**로 세부 항목 시각화 가능    |
| **장점**       | 작은 변동도 쉽게 인식 가능                | 구간별 절댓값, 항목별 구성 비율 명확히 표현 가능  |
| **단점**       | 데이터가 많고 변동이 심하면 **복잡해질 수 있음**  | 시간 간격이 좁은 경우 **해석 어려움**       |

## 4. 비교 시각화
### 히트맵 
- 그룹별 비교 요소가 많을 때 효과적인 시각화 방식
- 각 셀의 색상으로 데이터의 크고 작음을 표현하여 시각적으로 비교 가능
- 행은 그룹, 열은 요소를 의미하며, 각 그룹의 요소별 값을 한눈에 비교 가능
- 상관계수 히트맵처럼 변수 간 관계나 수치를 색상으로 표현할 수 있음
- 열을 시간 흐름으로 정하면 시간 시각화도 가능

### 방사형 차트
- RPG 게임 캐릭터 능력치를 표현하는 다변량 비교 시각화 방식
- 하나의 그룹이 여러 **변수(능력치)**를 가질 때 각 항목의 크기를 시각적으로 표현
- 방사형 축의 각 방향이 하나의 변수, 중심에서 멀수록 값이 큼
- 각 그룹의 능력 분포를 전체적으로 파악 가능

### 평행좌표 그래프
- 그룹별 변수 값을 선으로 연결해 시각화하는 방식
- 서로 다른 변수들의 스케일을 통일하여 비교 가능
- 변수 개수가 많고 분류가 복잡한 경우에도 유용하게 사용
- 데이터 전처리 과정에서 **정규화(0~100%)**를 하면 시각적 차이를 극대화 가능
- 각 그룹의 특성을 전체 변수 축 기준으로 파악 가능하며, 클러스터 간 패턴 차이 분석에 적합

## 5. 분포 시각화
### 연속형 변수
- **히스토그램**
  - 데이터를 구간(bin)으로 나눠 막대그래프로 표현
  - 구간 수에 따라 해석 난이도 변화 → 적절한 bin 수 선택이 중요

### 명목형 변수
- **파이차트 / 도넛차트**
  - 전체를 100%로 하여 비율을 면적(각도)로 표현
  - 도넛차트는 가운데 공간에 추가 정보 삽입 가능

### 복잡한 명목형 변수
- **트리맵**
  - 공간을 사각형으로 나눠 비율 표현
  - 세부 카테고리나 계층 구조 표현에 적합

### 원형 분포 강조
- **와플차트**
  - 격자 셀 단위로 비율 표현
  - 명확한 비교에 유리하나, 위계 구조는 표현 불가

## 6. 관계시각화

- 변수 간 관계를 시각적으로 파악하기 위한 방법.
- 보통 **산점도(Scatter Plot)**를 사용하여 두 변수의 관계를 표현.

### 산점도 (Scatter Plot)
- X축: 독립 변수, Y축: 종속 변수
- 점이 퍼진 모양을 통해 관계 유무, 강도, 방향 파악
- **극단값(이상치)**이나 **클러스터(집단)** 발견에 유용
- 점이 많을 경우 **과밀 문제** 발생 → 겹침으로 정보 손실 우려

### 버블 차트 (Bubble Chart)
- 산점도에 **제3의 변수**를 **원의 크기**로 추가 표현
- **색상**을 활용하면 제4의 변수까지 동시에 시각화 가능
- 시간 축을 활용하면 애니메이션처럼 **시간 변화에 따른 관계 변화**도 표현 가능

⚠️**관측치가 많을수록** 정보가 겹쳐 시각화 효율 떨어짐

## 7. 공간시각화
### 도트맵 (Dot Map)
- 동일한 크기의 점으로 특정 위치의 데이터 분포를 표현
- 단점: 값의 정량적 비교가 어려움
- **정확한 수치보단 분포 패턴 파악에 적합**

### 버블맵 (Bubble Map)
- **원 크기로 데이터 크기를 표현**
- 도트맵보다 정량적 비교에 유리
- 단점: 원이 겹칠 경우 해석이 어려울 수 있음

### 코로플레스맵 (Choropleth Map)
- **지역 단위(구·시·도 등)에 색상 농도로 값 표현**
- 상대적 비교에 효과적, 직관적
- **단점**: 인구 등 면적 차이로 왜곡 가능 → 주의 필요

### 커넥션맵 (Connection Map)
- 두 지점 간의 **이동·흐름·연결 관계**를 선으로 시각화
- 물류, 통신, 인구 이동 등에서 활용
